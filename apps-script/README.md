# Google Apps Script — Deployment Guide

This folder contains the **backend code** for the Car Owners Social Wall.
It must be deployed as a Google Apps Script Web App before the frontend can read or write posts.

## Prerequisites

- A Google account that owns the target Google Spreadsheet
  (`1t9tFiBdqYO74K-ysGrR-ButnteVX4qD7RcrFvGbQdQs`)

## Steps

### 1. Open Google Apps Script

Go to <https://script.google.com> and create a **New project**.

### 2. Paste the code

Delete any existing content in `Code.gs` and paste the entire contents of
[`Code.gs`](Code.gs) from this folder.

### 3. Save

Press **Ctrl+S** (or the 💾 icon) and give the project a name, e.g. *AdaniBackend*.

### 4. Deploy as Web App

1. Click **Deploy → New deployment**.
2. Click the ⚙️ gear icon → **Web app**.
3. Set:
   - **Execute as**: Me
   - **Who has access**: **Anyone** *(no sign-in required)*
4. Click **Deploy**.
5. Copy the **Web App URL** shown (it looks like
   `https://script.google.com/macros/s/<SCRIPT_ID>/exec`).

### 5. Configure the frontend

Open `docs/gasStore.js` and replace the `GAS_URL` placeholder with your
Web App URL:

```js
const GAS_URL = "https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec";
```

Save, commit, and push — GitHub Pages will serve the updated app.

### 6. Verify

Visit `<your-web-app-url>?action=health` in a browser. You should see:

```json
{"ok":true}
```

## Sheet structure

The script automatically creates a sheet tab named **`posts`** with these
columns if it does not already exist:

| id | plate | category | content | author | createdAt | hidden |
|----|-------|----------|---------|--------|-----------|--------|

- **id** — UUID generated by the script  
- **plate** — normalized digits-only plate number (7 or 8 digits)  
- **category** — `notice` / `warning` / `compliment` / `question`  
- **content** — post text (up to 1 000 characters)  
- **author** — optional display name  
- **createdAt** — ISO 8601 timestamp  
- **hidden** — set to `true` to hide a post from the public feed  

## API reference

| Method | Parameters | Description |
|--------|-----------|-------------|
| GET | `?action=health` | Returns `{"ok":true}` |
| GET | `?action=recent` | Returns up to 20 most-recent visible posts |
| GET | `?action=list&plate=XXXXXXX` | Returns all visible posts for a plate |
| POST | JSON body (see below) | Creates a new post |

### POST body

```json
{
  "plate":    "1234567",
  "category": "notice",
  "content":  "Your message here",
  "author":   "Optional name"
}
```

Response: `{"ok":true,"id":"<uuid>"}` or `{"ok":false,"error":"..."}`.
